## 🔍 What is Information Architecture?

**Information Architecture (IA)** is about how you organize, structure, and label content so users can find what they need quickly and easily. Think of it like the blueprint for a house—you don’t start building walls before deciding where the rooms, doors, and windows should go.

### Why It Matters

* Good IA = clarity, usability, and scalability
* Bad IA = confusion, frustration, and high support costs

### 🧠 Brain Teaser

> You’re documenting a simple app that does one thing: tells the time. But users in different time zones see different results. How would you structure the doc so each user knows how to get exactly what they need?

## 🧹 Topic-Based Authoring

### What is Topic-Based Authoring?

Topic-based authoring breaks content into small, focused, and reusable chunks called **topics**.

### Types of Topics

* **Concept**: Explains what something is or why it matters
* **Task**: Step-by-step guide to doing something
* **Reference**: Technical details like APIs or parameters

### Benefits

* Easier updates
* Content reuse
* Clear separation of concerns

### 🧠 Brain Teaser

> Imagine you're writing docs for a cooking robot. It can boil, fry, bake, and blend. How would you divide your content into concept, task, and reference topics to help a new user cook their first meal?

## 📂 Content Outlines, TOC Design & Modular Docs

### Step 1: Create a Content Outline

List all your content in logical order before writing.

### Step 2: Build a Table of Contents (TOC)

Use clear, structured navigation:

```markdown
- Getting Started
  - Introduction
  - Install the SDK
- Authentication
  - API Keys
  - OAuth 2.0
- Making Requests
  - Sample Request
  - Handling Errors
```

### Step 3: Think Modular

Use reusable content chunks across guides, wikis, and help centers.

### 🧠 Brain Teaser

> You’ve written five guides that all begin with the same two steps. Should you duplicate the steps in each guide, or modularize them into one shared section? Why?

## 🛍️ Navigation: TOCs, Breadcrumbs, and Headings

### TOCs

Your main menu. Keep it shallow, clear, and grouped.

### Breadcrumbs

Show users their location within the site structure.

```
Home > API Docs > Authentication > API Keys
```

### Headings

Use a consistent heading structure:

* `# H1` - Page title
* `## H2` - Section headers
* `### H3` - Sub-sections

### 🧠 Brain Teaser

> You’ve just written a huge tutorial. A user jumps straight to the middle. What do they need in the headings or layout to understand where they are and what came before?

## 🧠 User Journeys & Content Planning

### What is a User Journey?

A step-by-step map of how a user achieves a goal.

### Example:

A developer's journey:

1. Sign up
2. Get API Key
3. Make a request
4. Handle errors
5. Go live

Plan content for each step to match real workflows.

### 🧠 Brain Teaser

> Imagine a user has done everything right—except one small detail—and their integration fails. What could you add to your user journey doc to catch that one step they might miss?
